01 Uebersicht

ProCon ist eine webbasierte SaaS-Plattform fuer die zentrale Verwaltung von KI-System-Prompts. Teams koennen Prompts versionieren, freigeben und direkt ueber eine REST-API abrufen.

Was ProCon loest: Prompts landen oft in Slack-Nachrichten oder sind direkt im Code eingebettet. ProCon schafft eine einzige Quelle der Wahrheit fuer alle KI-Prompts.
RolleBeschreibung

DeveloperPrompts erstellen und bearbeiten

03 Erste Schritte

3.1 Registrierung

  1. /auth/register aufrufen
  2. Name, E-Mail und Passwort eingeben
  3. Nutzungsbedingungen akzeptieren
  4. "Konto erstellen" klicken → E-Mail bestaetigen

3.2 Erstes Projekt anlegen

  1. Im Dashboard auf "Neues Projekt" klicken
  2. Projektname und Beschreibung eingeben
  3. Projekt erscheint unter /dashboard/projects

3.3 Ersten Prompt erstellen

  1. Projekt oeffnen → "Neuer Prompt"
  2. Name, Slug und Inhalt einfüllen
  3. Status auf active setzen und speichern

3.4 API-Zugriff

  1. Zu /dashboard/profile navigieren
  2. API-Schluessel kopieren
  3. HTTP-Anfrage stellen:
GET /api/v1/prompts/{projekt-slug}/{prompt-slug}
Authorization: Bearer {API_KEY}
X-Schema: promptcontrol_prod

04 How-to Guides

4.1 Versionen & Rollback

  1. Prompt oeffnen → Seitenleiste Versionen
  2. Version auswaehlen → "Rollback" klicken
  3. Prompt wird zurueckgesetzt (neue Version angelegt)

4.2 Compliance-Scanner

Der Scanner prueft Prompt-Inhalte auf DSGVO-kritische Muster (E-Mail, Telefon, IBAN, Ausweisnummern).

  1. Prompt oeffnen → "Compliance-Check"
  2. Scanner analysiert und meldet kritische Stellen
Hinweis

Der Scanner ersetzt keine rechtliche Pruefung.

4.3 KI-Analyse

  1. Prompt oeffnen → "KI-Analyse starten"
  2. Verbesserungsvorschlaege erscheinen unterhalb des Editors

4.4 Export als instruction.md

  1. Prompt oeffnen → "Exportieren"
  2. Format waehlen: instruction.md oder skill.md

4.5 Teamverwaltung

  1. Zu /dashboard/team navigieren (Pro-Plan)
  2. "Mitglied einladen" klicken
  3. E-Mail und Rolle auswaehlen

4.7 Adminbereich

Nur fuer Nutzer mit Rolle Admin. Drei Bereiche:

  • Nutzerverwaltung (/dashboard/admin/users) - Rollen aendern, Nutzer entfernen
  • Aboverwaltung (/dashboard/admin/subscriptions) - Plan und Abo-Status aendern
  • App-Einstellungen (/dashboard/admin/settings) - Feature-Flags, Limits
Zugriff

Admin-Link erscheint in der Sidebar automatisch bei Rolle admin.

4.8 Profileinstellungen

Unter /dashboard/profile verfuegbar:

  • Persoenliche Informationen - Name aktualisieren
  • Passwort aendern - Neues Passwort setzen
  • API-Schluessel - Anzeigen, kopieren, neu generieren
  • Benachrichtigungen - E-Mail-Praeferenzen
  • Gefahrenzone - Konto loeschen

07 Compliance & DSGVO

  • Daten in der EU gehostet (Supabase EU-Region)
  • Passwort-Hashing via Supabase Auth (bcrypt)
  • API-Keys: 64-stellige Hex-Strings (256-bit Entropie)
  • GDPR-Consent-Banner bei erstem Besuch
  • Rechtliches unter /legal/impressum, /legal/datenschutz, /legal/agb
  • Compliance-Scanner prueft Prompt-Inhalte auf PII
